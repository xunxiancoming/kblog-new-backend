# 博客后端需求文档

用monorepo, 前后端分离，后端代码所在的目录叫做api，根据接口生成一个管理后台的页面，目录叫做admin。

## api项目需求
### 技术栈
1. nest.js
2. mysql
3. prisma
4. swagger
5. redis

需要用上面的技术栈实现以下接口：

1. 博客文章的增、删、改、查，需要鉴权，游客只能查看 ，管理者可以增、删、改。
2. 个人信息的增、删、改、查，需要鉴权，游客只能查看 ，管理者可以增、删、改。
3. 作品的增、删、改、查，需要鉴权，游客只能查看 ，管理者可以增、删、改。
4. 评论的增、删、改、查，需要鉴权，游客可以增，查，管理者可以删。文章和评论是一对多的关系，一篇文章对应多条评论。
5. 文章增加草稿功能。
6. 标签的增、删、改、查，需要鉴权，游客只能查看 ，管理者可以增、删、改。标签和文章是多对多的关系，即一篇文章可以有多个标签，一个标签可以有多篇文章。
8. 阅读数用redis来做。
7. 上面所有的接口需要生成swagger文档。

### 后台管理（admin）需求
## 技术栈
1. react
2. typescript
3. antd
4. markdown编辑器

需要实现以下功能：
1. 注册，登录，退出，注册需要用户名，密码，邮箱，重复密码，登录需要用户名，密码，记住密码，（7天内登录过，不需要再次登录）
2. 文章管理页，实现文章的增、删、改、查
3. 标签管理页，实现标签的增、删、改、查
4. 作品管理页，实现项目的增、删、改、查
5. 个人信息管理页，实现个人信息的增、删、改、查
6. 评论管理页，实现评论的增、删、改、查
7. 统计管理页，用图表展示文章数量，评论数量，作品数量，标签数量，以及文章阅读数量，按月统计，显示趋势图。
